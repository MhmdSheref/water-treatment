API Reference

ReWater — complete endpoint documentation with integration examples.

http://localhost:3000
Authentication: All endpoints except /api/auth/login and /api/health require a JWT Bearer token.
Obtain a token from POST /api/auth/login and include it as:
Authorization: Bearer <token>
General
GET
/api/health
Server health check. Returns sensor mode and collection interval. No auth required.
Example Request
fetch('/api/health')
  .then(r => r.json())
  .then(console.log);
Response
{
  "status": "ok",
  "timestamp": "2026-03-18T12:00:00.000Z",
  "sensorMode": "placeholder",
  "sensorIntervalMs": 30000
}
Authentication
POST
/api/auth/login
Obtain a JWT token. Token expires in 24 h.
Request Body
FieldTypeRequiredDescription
usernamestring✓User's username
passwordstring✓User's password
Example Request
const res = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: 'admin123' })
});
const { token, user } = await res.json();
// Store token for subsequent requests
Response
{
  "token": "eyJhbGci...",
  "user": { "id": 1, "username": "admin", "role": "admin" }
}
GET
/api/auth/me any role
Returns the currently authenticated user's profile.
Example Request
const res = await fetch('/api/auth/me', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const { user } = await res.json();
Technician Endpoints
GET
/api/tech/dashboard technician · admin
Live plant status: latest sensor reading, WQI score, and compliance flags. Also persists compliance log entries.
Example Request
const data = await fetch('/api/tech/dashboard', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());

console.log(data.wqi.score);         // e.g. 87
console.log(data.compliance.overall); // true / false
console.log(data.reading.BOD);        // mg/L
Response Shape
{
  "timestamp": "2026-03-18T12:00:00Z",
  "reading": { "pH": 7.2, "BOD": 45, "COD": 90, "TSS": 38, /* ... */ },
  "compliance": {
    "overall": true,
    "nonCompliantCount": 0,
    "flags": { "BOD": false, "COD": false /* ... */ }
  },
  "wqi": { "score": 87, "interpretation": "Good" }
}
GET
/api/tech/readings technician · admin
Recent sensor readings in descending order (newest first). Useful for listing history.
Query Parameters
ParamDefaultDescription
hours24How many hours back to fetch
limit100Maximum rows returned
Example
// Last 6 hours, max 50 rows
const { readings } = await fetch('/api/tech/readings?hours=6&limit=50', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());
GET
/api/tech/readings/history technician · admin
Readings in ascending chronological order — designed for feeding directly into Chart.js or time-series graphs.
Query Parameters
ParamDefaultDescription
hours1Timespan in hours
limit120Max data points
Chart.js Integration Example
const { readings } = await fetch('/api/tech/readings/history?hours=6&limit=120', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());

new Chart(ctx, {
  type: 'line',
  data: {
    labels: readings.map(r => new Date(r.timestamp).toLocaleTimeString()),
    datasets: [{
      label: 'BOD (mg/L)',
      data: readings.map(r => r.BOD)
    }]
  }
});

Admin Endpoints
POST
/api/admin/simulate admin
Runs the full analysis pipeline (Deviation, WQI, Reactor Sizing, Energy, MILP Optimization) on provided plant data. Result is persisted to the database.
Request Body — Plant Parameters
FieldUnitDescription
BODmg/LBiochemical Oxygen Demand
CODmg/LChemical Oxygen Demand
TSSmg/LTotal Suspended Solids
TNmg/LTotal Nitrogen
TPmg/LTotal Phosphorus
ECdS/mElectrical Conductivity
Namg/LSodium
heavy_metalsmg/LHeavy metals concentration
flow_ratem³/hPlant flow rate
Example Request
const result = await fetch('/api/admin/simulate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    BOD: 55, COD: 110, TSS: 50, TN: 35, TP: 9,
    EC: 3.2, Na: 250, heavy_metals: 0.12, flow_rate: 500
  })
}).then(r => r.json());

console.log(result.wqi.wqi);                      // WQI score
console.log(result.deviation.nonCompliantParams);  // ['BOD', 'TSS']
console.log(result.optimization.selectedUpgrades);  // MILP upgrades
console.log(result.energy.total_power_kW);          // kW
Response Shape
{
  "simulationId": "uuid",
  "wqi": { "wqi": 112, "interpretation": "Poor" },
  "deviation": { "nonCompliantParams": ["BOD"], "deviations": { "BOD": 15 } },
  "sizing": { "biological": { "hrt_hours": 3.22, "reactor_volume_m3": 1610 } },
  "energy": { "total_power_kW": 24.8, "sec_kWh_per_m3": 0.05 },
  "optimization": { "feasible": true, "selectedUpgrades": [ /* ... */ ] },
  "recommendations": [ /* urgency-sorted array */ ]
}
GET
/api/admin/simulations admin
List all past simulation runs with WQI and input snapshot.
Query Parameters
ParamDefaultDescription
limit50Max rows
offset0Pagination offset
Example
const { simulations, total } = await fetch(
  '/api/admin/simulations?limit=10&offset=0',
  { headers: { 'Authorization': `Bearer ${token}` } }
).then(r => r.json());

for (const sim of simulations) {
  const result = JSON.parse(sim.result_json);
  console.log(sim.id, result.wqi.wqi);
}
GET
/api/admin/simulations/:id admin
Full detail for a single simulation, including parsed input and result objects.
Example
const sim = await fetch(`/api/admin/simulations/${simId}`, {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());
// sim.input  → original plant data
// sim.result → full analysis pipeline output
POST
/api/admin/recalculate admin
"What-if" analysis — runs the full pipeline with optional config overrides (different standards or weights). Does not persist a simulation record.
Example — Override BOD standard
const result = await fetch('/api/admin/recalculate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    plantData: { BOD: 55, COD: 110, TSS: 50, flow_rate: 500 /* ... */ },
    configOverrides: {
      standards: { BOD: 50 }  // temporarily relax standard to 50 mg/L
    }
  })
}).then(r => r.json());
GET
/api/admin/readings admin
Full sensor data for any date range, ascending order. Includes min/avg/max statistics per parameter. Supports up to 5000 rows.
Query Parameters
ParamDefaultDescription
from—ISO 8601 start datetime
tonowISO 8601 end datetime
hours24Fallback if from/to not set
limit1000Max rows (hard cap 5000)
Example — Date range query
const qs = new URLSearchParams({
  from: '2026-03-18T00:00:00Z',
  to:   '2026-03-18T23:59:59Z',
  limit: '500'
});
const { readings, stats } = await fetch(`/api/admin/readings?${qs}`, {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());

console.log(stats.BOD);  // { min: 32, avg: 46.1, max: 67 }
GET
/api/admin/logs admin
System event log — logins, simulations, recalculations, config changes. Filterable by event type and date range.
Query Parameters
ParamDescription
typeLOGIN · SIMULATION · RECALCULATION · CONFIG_UPDATE
fromISO 8601 start date filter
toISO 8601 end date filter
limitMax rows (default 100)
offsetPagination offset (default 0)
Example
// Get all simulations performed today
const { events } = await fetch(
  '/api/admin/logs?type=SIMULATION&limit=50',
  { headers: { 'Authorization': `Bearer ${token}` } }
).then(r => r.json());
GET
/api/admin/config admin
Returns all system configuration: regulatory standards, AHP weights, kinetic constants, optimization weights.
Example
const { config } = await fetch('/api/admin/config', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());
// config → [{ key: 'standards_BOD', value: '40', description: '...' }, ...]
Key Prefixes
PrefixDescription
standards_Regulatory limits (Egyptian Code)
lod_Limits of Detection
weight_AHP weights for WQI (must sum to 1)
k_Kinetic constants
membrane_UF/RO membrane flux values
opt_MILP optimization weights (α, β, γ)
PUT
/api/admin/config admin
Update one or more configuration values. Change is persisted and logged in the event log.
Example — Tighten BOD standard & adjust weight
await fetch('/api/admin/config', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    standards_BOD: '30',       // tighten BOD limit to 30 mg/L
    weight_BOD: '0.25',        // increase BOD weight in WQI
    opt_alpha: '0.5'           // increase cost weight in MILP
  })
});
